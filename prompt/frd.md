## 目标
实现一个企业内部使用的LLM API Gateway平台。

## 主要功能：
为企业内部员工提供OpenAPI，以Gateway方式中转到供应商的LLM API服务，以便他们配置在喜欢的Coding IDE或插件或CLI 工具中使用vibe coding的能力。

## 模块规划：

- 核心模块
  - 平台登录
    - 是否有UI：有UI
    - 用户角色：系统管理员
      - 主要功能：
        - 1. 初始化系统管理员：在部署系统配置文件（如 .env 文件）初始化管理员企业邮箱地址，系统第一次启动（连接数据库）时自动向用户管理表插入一条记录，包括管理员企业邮箱地址、用户角色初始化为系统管理员等
    - 用户角色：系统管理员、团队管理者、普通用户
      - 主要功能：
        - 1. 用户登录：系统管理员、团队管理者、普通用户在登录界面输入用户企业邮箱，系统通过企业邮箱域名后缀校验后，后台生成随机密码发送企业邮箱，用户再输入随机密码登录系统
        - 2. 账号状态校验：登录时系统自动校验用户是否被管理员禁用
  - 平台管理
    - 是否有UI：有UI
    - 用户角色：系统管理员
      - 主要功能：
        - 1. LLM服务供应商管理：包括供应商名称，LLM API Base URL，API密钥，API说明等，可增删改查
        - 2. 模型列表管理：包括供应商名称，模型名称，计费单价（输入/输出每百万Token价格）等，可增删改查
        - 3. 企业OpenAPI管理：配置对内暴露的Gateway API端点，包括API名称，API类型（如兼容ChatGPT API、Anthropic API等），API地址（Gateway URL），API用法说明等，可增删改查
        - 4. 用户管理：包括用户企业邮箱，用户角色（系统管理员、团队管理者、普通用户），API消耗费用限制（默认继承角色，支持针对单独用户进行自定义修改），账号状态（启用/禁用），可增删改查
        - 5. 角色管理：包括角色名称，权限（如工作台、数据分析、平台管理），API消耗费用限制（系统默认值），API Key有效时长等，可增删改查
        - 6. API Key列表管理：企业内部用户申请的所有API Key列表，包括用户企业邮箱，Key名称，申请日期，Key有效期、状态（启用/禁用）等，只可延长Key有效期
        - 7. 额度恢复策略设置：定义费用限制的刷新周期（如每月1号自动重置、不自动重置等）
  - 工作台
    - 是否有UI：有UI
    - 用户角色：普通用户、团队管理者、系统管理员
      - 主要功能：
        - 1. 申请API Key：用户为当前账号申请API Key，填写Key名称，选择到期时间（不可超过角色限制的有效时长）。后台生成唯一Key并直接在UI展示（仅展示一次），不再重复发送邮件。一个企业邮箱可申请多个API Key
        - 2. 查看API Key：关联当前登录用户，查看已申请的API Key列表，包括Key名称、Key掩码（如sk-...abcd）、申请日期、有效期、状态（启用/禁用）
        - 3. 启禁/删除API Key：关联当前登录的用户企业邮箱管理已申请的API Key进行启禁/删除操作
        - 4. 模型列表查看：包括供应商名称，模型名称，单价价格等，只可查看
        - 5. 企业OpenAPI查看：用来作为API Gateway中转到供应商的LLM API，包括API名称，API类型（如兼容ChatGPT API、Anthropic API等），API地址（Gateway URL），API用法说明等，只可查看
  - API服务
    - 是否有UI：没有UI，纯API服务
    - 用户角色：无关
      - 主要功能：
        - 1. API Gateway：对企业内部用户暴露OpenAPI（兼容ChatGPT API、Anthropic API等主流大模型服务格式），接收请求并拦截校验API Key的合法性、有效性、用户账号状态以及当前周期的费用额度是否超限，校验通过后作为Gateway将请求中转到供应商的LLM API，并将返回结果透传回来
        - 2. 额度实时扣减：根据每次请求大模型返回的实际Token数量（输入/输出）及对应模型单价，实时计算接口费用，并同步扣减该用户的当前周期剩余额度
        - 3. 日志记录：将每一次的请求进行明细日志记录，包括API Key、时间、模型、输入输出Token数量、用户Prompt及返回结果等 
        - 4. 高并发服务：支持高并发、高可用的OpenAI/Anthropic等主流格式的路由和中转访问服务
  - 数据分析
    - 是否有UI：有UI，数据可视化
    - 用户角色：系统管理员、普通用户、团队管理者
      - 主要功能：
        - 1. 消耗统计：关联当前登录的用户企业邮箱查看自身API Key的LLM Token、接口费用等，支持按照日、周、月以及截止当前累积等不同时间周期、不同模型的数据统计、趋势分析等；展示当前周期已消耗总和、额度使用比例（当前周期已消耗总和与费用限制比）等
        - 2. 消耗排名：以用户（企业邮箱）为汇总粒度，查看不同周期（日、周、月以及截止当前累积等）、不同模型下的LLM Token、接口费用统计数据和排名。隐去具体用户信息（企业邮箱），只展示Top 30%的用户和当前登录用户数据
        - 3. 提示词分类统计：对提示词进行自动分类，展示日、周、月以及截止当前累积等不同时间周期下，提示词分类按照LLM Token、接口费用等统计数据排名和趋势分析等
    - 用户角色：系统管理员、团队管理者
      - 主要功能：
        - 1. 消耗明细：支持按用户、按时间周期（日、周、月以及截止当前累积）、按不同模型下钻查询LLM Token、接口费用的统计数据、趋势分析等
        - 2. 消耗汇总：全体用户汇总在日、周、月不同时间周期下、不同模型下的总LLM Token、接口费用统计数据和趋势波动分析等
