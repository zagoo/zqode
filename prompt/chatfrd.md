# AI Chat功能需求文档【FRD】

## 目标

给企业内部使用的 LLM API Gateway 平台新增一个 AI Chat 界面。

AI Chat 用于让企业内部员工在平台 UI 中，基于自己已申请或被授权使用的 Key，选择可用模型并直接与 AI 进行对话。所有对话请求仍统一经过 LLM API Gateway，不绕过平台现有的鉴权、模型权限、限流、审计和用量统计能力。

## 主要功能：

可以让企业内部员工使用自己在 LLM API Gateway 平台申请或被授权的 Key，在 UI 中直接与 AI 沟通对话，并支持会话历史、多轮上下文和基础调用记录。

## 模块规划：

- 核心模块
  - AI Chat
    - 是否有UI：有UI
    - 用户角色：系统管理员、团队管理者、普通用户
      - 主要功能：
        - 1. Key选择
          - 可以从当前用户已申请或被授权的 Key 列表选择一个 Key
          - Key 列表只展示 Key 名称、所属团队、状态、可用额度、可用模型等摘要信息
          - 不在 AI Chat 页面展示 Key 明文
          - 已禁用、已过期、无额度或无模型权限的 Key 不可选择，并提示不可用原因
        - 2. 模型选择
          - 可以从平台提供且当前 Key 有权限调用的模型列表中选择一个模型
          - 模型列表应根据所选 Key 的权限动态过滤
          - 模型不可用、被下线或当前 Key 无权限时，不允许发起对话
          - 模型选择只影响后续请求，历史消息不因切换模型而自动重写
        - 3. AI对话
          - 可以输入文本提示词并接收模型返回结果
          - 每次请求必须通过 LLM API Gateway 发起，不允许前端直接请求模型供应商
          - 请求使用当前选择的 Key 和模型，并按平台现有规则进行鉴权、限流、用量统计和错误处理
          - 支持基础对话状态展示，包括生成中、生成完成、生成失败
          - 生成失败时展示失败原因，用户可以重新发送当前消息
        - 4. 会话历史与多轮对话
          - 支持新建会话
          - 支持查看历史会话
          - 支持加载历史会话并继续对话
          - 支持删除自己的历史会话
          - 多轮对话以上下文消息列表为基础，不单独设计额外的“对话状态保存”功能
          - 会话历史应保存消息内容、使用的 Key 标识、模型标识、创建时间、更新时间和调用状态
          - 会话历史中不得保存 Key 明文
          - 当会话上下文超过模型限制时，系统按平台策略截断较早消息或提示用户新建会话
        - 5. 图片对话
          - 当所选模型支持图片输入时，才展示图片上传入口
          - 支持用户在单次对话中上传图片，并基于图片内容向模型提问
          - 图片能力只用于当前会话上下文，不扩展为知识库、文件管理系统或跨会话文件复用
          - 图片类型、大小、数量、保存时长按照平台统一配置控制
          - 不支持的图片应明确提示原因
        - 6. 调用记录与用量统计
          - AI Chat 发起的请求纳入 LLM API Gateway 现有调用记录
          - 用量统计复用平台已有能力，不在 AI Chat 中新增报表模块
      - 页面组成：
        - 1. Key 和模型选择区
        - 2. 历史会话列表区
        - 3. 对话内容展示区
        - 4. 消息输入区
        - 5. 图片上传区
        - 6. 错误提示区
## 关键规则：

- 权限规则
  - 用户只能选择自己有权限的 Key
  - 用户只能选择当前 Key 有权限调用的模型
  - 不同用户之间的会话历史默认隔离
  - 不同团队之间的会话和用量数据默认隔离
- 请求规则
  - AI Chat 所有请求必须复用 LLM API Gateway 现有请求链路
  - AI Chat 不新增独立模型调用通道
  - AI Chat 不绕过现有限流、熔断、日志、计费/统计和审计规则
- 会话规则
  - 一个会话可以包含多轮消息
  - 历史会话继续对话时，默认沿用最近一次请求使用的 Key 和模型
  - 用户切换 Key 或模型时，只影响后续请求，并需要重新校验权限
  - 当 Key 或模型不可用时，历史会话可以查看，但不能继续用该 Key 或模型发起新请求
  - 用户删除会话后，前台不再展示；后台数据保留策略遵循平台统一数据留存配置

## 验收标准：

- 普通用户可以进入 AI Chat 页面，并只看到自己有权限使用的 Key
- 选择 Key 后，只展示该 Key 有权限调用的模型
- 用户可以完成一次文本对话，并收到模型返回结果
- 用户可以在同一会话中连续提问，模型可以基于当前会话上下文响应
- 用户可以关闭页面后重新打开历史会话并继续对话
- 用户无法查看其他用户的历史会话
- 已禁用、过期、超额或无模型权限的 Key 无法用于发起新请求
- AI Chat 请求可以由系统管理员、团队管理者在分析模块的消耗明细中查询到
- 图片上传入口只有在模型支持时才出现
